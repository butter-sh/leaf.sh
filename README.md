# leaf.sh

🌿 Beautiful documentation site generator for arty.sh projects

## Overview

leaf.sh is a powerful bash script that transforms your arty.sh project into a stunning static documentation website. It automatically parses your project's structure, including README, source files, examples, and metadata, and generates a modern, responsive documentation site with:

- 🎨 **Modern Design** - Built with Tailwind CSS v4
- 🌓 **Dark/Light Theme** - Toggle between themes with localStorage persistence
- 💻 **Syntax Highlighting** - Beautiful code highlighting with Highlight.js
- 📱 **Responsive Layout** - Mobile-first design that works everywhere
- ⚡ **Fast & Lightweight** - Single HTML file with CDN assets
- 🎯 **Zero Config** - Works out of the box with any arty.sh project

## Features

### Automatic Content Detection

The generator automatically scans and includes:
- Project metadata from `arty.yml`
- Main documentation from `README.md`
- Source files with syntax highlighting
- Example files from `examples/` directory
- Project icon from `_assets/icon/`
- License information

### Beautiful UI Components

- Gradient hero section with floating animations
- Card-based layout for files
- Smooth scroll navigation
- Hover effects and transitions
- Accessible color schemes

### Developer-Friendly

- Language auto-detection from file extensions
- Support for bash, JavaScript, Python, and more
- Proper code formatting and indentation
- Copy-paste friendly code blocks

```bash
hammer leaf my-docs-generator

```

## Usage

### Basic Usage

Generate documentation for the current directory:

```bash
./leaf.sh
```

### Specify Project Directory

Generate documentation for a specific project:

```bash
./leaf.sh /path/to/your/arty-project
```

### View Help

```bash
./leaf.sh --help
```

## Example

Generate documentation for an arty.sh project:

```bash
# Navigate to your arty.sh project
cd ~/projects/my-bash-library

# Generate documentation
~/leaf.sh/leaf.sh

# Open the generated site
open docs/index.html
```

The documentation will be created in the `docs/` directory of your project.

## Requirements

- Bash 4.0 or higher
- An arty.sh compatible project structure

## Output Structure

```
your-project/
└── docs/
    └── index.html    # Complete documentation site
```

The generated `index.html` is a self-contained file that includes:
- All documentation content
- External CDN links for styling and functionality
- Responsive design and dark mode
- Interactive navigation

## Project Structure

For best results, structure your arty.sh project like this:

```
your-project/
├── arty.yml           # Project metadata
├── README.md          # Main documentation
├── LICENSE            # License file
├── main.sh            # Source files
├── utils.sh           # More source files
├── _assets/
│   └── icon/
│       └── icon.svg   # Project icon
└── examples/
    └── usage.sh       # Example files
```

## Customization

The generated HTML uses:
- **Tailwind CSS v4** - For styling
- **Highlight.js** - For syntax highlighting (Atom One Dark theme)
- **Marked.js** - For markdown parsing
- **Inter font** - For UI text
- **JetBrains Mono** - For code blocks

To customize, you can modify the generated `index.html` file or update the template in `leaf.sh`.

## Tips

1. **Add a good README** - The overview section pulls from your README.md
2. **Use descriptive filenames** - They appear in the navigation
3. **Add an icon** - Place an SVG icon in `_assets/icon/`
4. **Write examples** - Put usage examples in `examples/` directory
5. **Fill arty.yml** - Complete metadata enhances the documentation
