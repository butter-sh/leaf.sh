# leaf.sh

🌿 Beautiful documentation site generator and landing page builder for bash projects

## Overview

leaf.sh is a powerful dual-purpose bash script that:

1. **Transforms arty.sh projects into stunning static documentation websites** with modern design, syntax highlighting, and dark/light theme support
2. **Generates beautiful landing pages** for butter.sh with a cool, urban military carbon minimalistic theme

**New in v2.1:** Uses `yq` for reliable YAML parsing, improved help function, better error handling, and debug mode.

## Features

### Documentation Generator Mode

- 🎨 **Modern Design** - Built with Tailwind CSS
- 🌓 **Dark/Light Theme** - Toggle between themes with localStorage persistence
- 💻 **Syntax Highlighting** - Beautiful code highlighting with Highlight.js (20+ languages)
- 📱 **Responsive Layout** - Mobile-first design that works everywhere
- ⚡ **Fast & Lightweight** - Single HTML file with CDN assets
- 🎯 **Configurable** - Custom logo path and HTML base-path support
- 🏠 **Ecosystem Link** - Integrated link to butter.sh organization
- 🔧 **YAML Parsing** - Powered by `yq` for robust YAML parsing

### Landing Page Generator Mode

- 🎨 **Carbon Theme** - Cool, stylish urban military aesthetic
- 🎭 **Theme Switcher** - Smooth fade between dark and light modes using iconset.css icons
- 📦 **Project Cards** - Display multiple projects with descriptions
- 🎯 **Dual CTAs** - Primary gradient CTA and secondary outline CTA
- 📱 **Responsive** - Mobile-friendly header with navigation
- 🔗 **GitHub Integration** - Configurable GitHub organization link with icons
- 🖼️ **Brand Integration** - Uses butter.sh cube-carbon-light.svg branding
- 🎨 **Light Green Accent** - Primary color: `#86efac` → `#4ade80`

## Prerequisites

### Required Dependencies

- **yq** - YAML parser (v4.x recommended)
  ```bash
  # macOS
  brew install yq
  
  # Linux (snap)
  snap install yq
  
  # Linux (wget)
  wget https://github.com/mikefarah/yq/releases/latest/download/yq_linux_amd64 -O /usr/local/bin/yq
  chmod +x /usr/local/bin/yq
  
  # Or see: https://github.com/mikefarah/yq#install
  ```

### Optional Dependencies

- `find`, `sed`, `grep` - Standard Unix tools (usually pre-installed)
- Web browser - For viewing generated documentation

## Usage

### Quick Start

```bash
# Generate docs for current directory
./leaf.sh

# Show help
./leaf.sh --help

# Show version
./leaf.sh --version
```

### Documentation Mode

#### Basic Usage

Generate documentation for the current directory:

```bash
./leaf.sh
```

Generate documentation for a specific project:

```bash
./leaf.sh /path/to/project
```

#### Custom Logo

Generate documentation with a custom logo:

```bash
./leaf.sh --logo /path/to/icon.svg
```

The script will automatically search for logos in this order:
1. Custom logo specified with `--logo`
2. `_assets/icon/icon.svg`
3. `_assets/icon/icon-v2.svg`
4. `_assets/icon/icon-simple.svg`
5. `_assets/icon/icon.png`
6. Default fallback icon

#### Custom Output Directory

```bash
./leaf.sh -o ./public
./leaf.sh --output /var/www/html/docs
```

#### Custom Base Path

For deployment to subdirectories:

```bash
./leaf.sh --base-path /docs/
./leaf.sh --base-path /projects/myproject/
```

#### Full Example

```bash
./leaf.sh /path/to/project \
  --logo ./custom-icon.svg \
  --base-path /docs/ \
  --github https://github.com/my-org \
  -o ./output
```

### Landing Page Mode

#### Basic Landing Page

Generate a butter.sh landing page:

```bash
./leaf.sh --landing
```

#### Custom GitHub Link

```bash
./leaf.sh --landing --github https://github.com/my-org
```

#### Custom Projects

Provide a JSON array of projects to display:

```bash
./leaf.sh --landing --projects '[
  {
    "url": "https://hammer.sh",
    "label": "hammer.sh",
    "desc": "Configurable bash project generator",
    "class": "card-project"
  },
  {
    "url": "https://arty.sh",
    "label": "arty.sh",
    "desc": "Bash library repository manager",
    "class": "card-project"
  },
  {
    "url": "https://leaf.sh",
    "label": "leaf.sh",
    "desc": "Beautiful documentation generator",
    "class": "card-project"
  }
]'
```

#### Custom Logo for Landing

```bash
./leaf.sh --landing --logo /path/to/brand/logo.svg
```

#### Complete Landing Example

```bash
./leaf.sh --landing \
  --logo ./butter.sh/_assets/brand/cube-carbon-light.svg \
  --github https://github.com/butter-sh \
  --base-path / \
  --projects '[...]' \
  -o ./public
```

### Debug Mode

Enable debug output for troubleshooting:

```bash
./leaf.sh --debug
./leaf.sh --debug /path/to/project
```

## Command Line Options

| Option | Description | Default |
|--------|-------------|---------|
| `--landing` | Generate landing page instead of docs | Documentation mode |
| `--logo PATH` | Path to logo/icon file | Auto-detect from `_assets/icon/` |
| `--base-path PATH` | Base path for HTML links | `/` |
| `--github URL` | GitHub organization URL | `https://github.com/butter-sh` |
| `--projects JSON` | JSON array of projects (landing mode) | Default butter.sh projects |
| `-o, --output DIR` | Output directory | `docs/` |
| `-h, --help` | Show help message | - |
| `--version` | Show version information | - |
| `--debug` | Enable debug output | Disabled |

## Project Structure

### Recommended arty.sh Project Structure

For best results with documentation mode, structure your arty.sh project like this:

```
your-project/
├── arty.yml           # Project metadata (parsed with yq)
├── README.md          # Main documentation (rendered as markdown)
├── LICENSE            # License file
├── main.sh            # Main source file
├── utils.sh           # Additional source files
├── lib/
│   └── helpers.sh     # Library files
├── _assets/
│   └── icon/
│       ├── icon.svg       # Project icon (SVG preferred)
│       └── icon-v2.svg    # Alternative icon
└── examples/
    ├── basic.sh       # Basic usage example
    └── advanced.sh    # Advanced usage example
```

### arty.yml Format

The `arty.yml` file should contain:

```yaml
name: "project-name"
version: "1.0.0"
description: "Project description"
author: "Your Name"
license: "MIT"
main: "main.sh"

scripts:
  start: "bash main.sh"
  test: "bash test.sh"
```

**Note:** leaf.sh uses `yq` to parse this file, ensuring robust and accurate YAML parsing.

## Examples

### Generate Docs for arty.sh Project

```bash
cd ~/projects/my-arty-project

# Basic generation
~/leaf.sh/leaf.sh

# With custom settings
~/leaf.sh/leaf.sh --logo ./_assets/icon/icon.svg --base-path /docs/

# Open in browser
open docs/index.html
```

### Generate butter.sh Landing Page

```bash
cd ~/projects/butter.sh

# Generate landing page
~/leaf.sh/leaf.sh --landing \
  --logo ./_assets/brand/cube-carbon-light.svg \
  --github https://github.com/butter-sh

# Open in browser
open docs/index.html
```

### Deploy to GitHub Pages

```bash
# Generate docs with proper base path
./leaf.sh --base-path /my-project/ -o docs

# Commit and push
git add docs/
git commit -m "Update documentation"
git push origin main

# Enable GitHub Pages from docs/ folder in repository settings
```

### CI/CD Integration

```yaml
# .github/workflows/docs.yml
name: Generate Documentation

on:
  push:
    branches: [ main ]

jobs:
  docs:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Install yq
        run: |
          sudo wget https://github.com/mikefarah/yq/releases/latest/download/yq_linux_amd64 -O /usr/local/bin/yq
          sudo chmod +x /usr/local/bin/yq
      
      - name: Generate Documentation
        run: |
          chmod +x leaf.sh
          ./leaf.sh --base-path /${{ github.event.repository.name }}/
      
      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./docs
```

## Tips & Best Practices

### Documentation Mode

1. **Use Comprehensive README** - Add a detailed README.md for better overview sections
2. **Descriptive Filenames** - Use clear filenames for better navigation clarity
3. **Add SVG Icons** - Include an SVG icon in `_assets/icon/` for branding
4. **Practical Examples** - Write useful examples in `examples/` directory
5. **Complete Metadata** - Fill all fields in `arty.yml` for rich metadata
6. **Code Comments** - Well-commented code looks better in documentation
7. **Markdown Formatting** - Use proper markdown in README for best rendering

### Landing Page Mode

1. **High-Contrast Descriptions** - Use clear, concise project descriptions
2. **Keep It Focused** - Don't overwhelm with too many projects (3-6 is ideal)
3. **Test Both Themes** - Check appearance in both dark and light modes
4. **Verify Links** - Ensure all GitHub and project URLs are correct
5. **Brand Consistency** - Use the butter.sh brand icon for consistency
6. **Mobile First** - Always test on mobile devices
7. **Fast Loading** - Keep project JSON simple for quick rendering

### Both Modes

1. **Test on Mobile** - Always check responsiveness on mobile devices
2. **Validate Links** - Test all links before deployment
3. **Use Proper Base Path** - Set correct base-path for subdirectory deployment
4. **Check HTML Output** - Validate generated HTML with browser dev tools
5. **Version Control** - Commit generated docs for easy rollback
6. **SEO Optimization** - Add proper meta tags if you customize the HTML
7. **Accessibility** - Ensure proper contrast ratios and semantic markup

## Troubleshooting

### Common Issues

#### yq not found

```bash
# Error: Missing required dependencies: yq
# Solution: Install yq
brew install yq  # macOS
# or
sudo snap install yq  # Linux
```

#### No project metadata

```bash
# Warning: arty.yml not found
# Solution: Create arty.yml with at least:
cat > arty.yml << EOF
name: "my-project"
version: "1.0.0"
description: "My project description"
EOF
```

#### Icon not showing

```bash
# Issue: Default icon appears instead of custom icon
# Solution: Place icon in correct location
mkdir -p _assets/icon
cp /path/to/icon.svg _assets/icon/icon.svg
```

#### Syntax highlighting not working

```bash
# Issue: Code appears without highlighting
# Solution: Check file extension is supported
# Supported: .sh, .bash, .js, .py, .rb, .go, .rs, etc.
# Or check browser console for errors
```

#### Base path issues

```bash
# Issue: CSS/JS not loading on deployed site
# Solution: Use correct base-path
./leaf.sh --base-path /my-project/
# Not: ./leaf.sh --base-path my-project
```

### Debug Mode

Use debug mode to troubleshoot issues:

```bash
./leaf.sh --debug

# Output will show:
# - Parsed YAML values
# - Found icon paths
# - Scanned files
# - Other diagnostic information
```

## Customization

Both generated HTML files are standalone and can be customized:

### Style Customization

Edit the `<style>` block in the generated HTML:

```css
/* Change primary gradient */
.gradient-text {
    background: linear-gradient(135deg, #your-color1 0%, #your-color2 100%);
}

/* Change dark mode background */
body.dark {
    background: linear-gradient(to-br, #your-dark-color1, #your-dark-color2);
}
```

### Add Custom JavaScript

Add to the `<script>` block:

```javascript
// Add smooth scroll behavior
document.querySelectorAll('a[href^="#"]').forEach(anchor => {
    anchor.addEventListener('click', function (e) {
        e.preventDefault();
        document.querySelector(this.getAttribute('href')).scrollIntoView({
            behavior: 'smooth'
        });
    });
});
```

### Include Additional CDN Libraries

Add to the `<head>` section:

```html
<!-- Add Chart.js for data visualization -->
<script src="https://cdn.jsdelivr.net/npm/chart.js"></script>

<!-- Add AOS for animations -->
<link href="https://unpkg.com/aos@2.3.1/dist/aos.css" rel="stylesheet">
<script src="https://unpkg.com/aos@2.3.1/dist/aos.js"></script>
```

### Extend with Meta Tags

For better SEO:

```html
<meta property="og:title" content="Your Project">
<meta property="og:description" content="Your description">
<meta property="og:image" content="https://your-site.com/og-image.png">
<meta name="twitter:card" content="summary_large_image">
```
