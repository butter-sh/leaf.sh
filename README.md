## Usage

### Documentation Mode

#### Basic Usage

Generate documentation for the current directory:

```bash
./leaf.sh
```

#### Custom Logo

Generate documentation with a custom logo:

```bash
./leaf.sh --logo /path/to/icon.svg
```

#### Custom Base Path

For deployment to subdirectories:

```bash
./leaf.sh --base-path /docs/
```

#### Full Example

```bash
./leaf.sh /path/to/project --logo ./custom-icon.svg --base-path /docs/ -o ./output
```

### Landing Page Mode

#### Basic Landing Page

Generate a butter.sh landing page:

```bash
./leaf.sh --landing
```

#### Custom GitHub Link

```bash
./leaf.sh --landing --github https://github.com/my-org
```

#### Custom Projects

```bash
./leaf.sh --landing --projects '[
  {
    "url": "https://hammer.sh",
    "label": "hammer.sh",
    "desc": "Configurable bash project generator",
    "class": "card-project"
  },
  {
    "url": "https://arty.sh",
    "label": "arty.sh",
    "desc": "Bash library repository manager",
    "class": "card-project"
  }
]'
```

#### Custom Logo for Landing

```bash
./leaf.sh --landing --logo /path/to/brand/logo.svg
```

#### Complete Landing Example

```bash
./leaf.sh --landing \
  --logo ./butter.sh/_assets/brand/cube-carbon-light.svg \
  --github https://github.com/butter-sh \
  --base-path / \
  --projects '[...]' \
  -o ./public
```

## Command Line Options

| Option | Description | Default |
|--------|-------------|---------|
| `--landing` | Generate landing page instead of docs | Documentation mode |
| `--logo PATH` | Path to logo/icon file | Auto-detect from `_assets/icon/` |
| `--base-path PATH` | Base path for HTML links | `/` |
| `--github URL` | GitHub organization URL | `https://github.com/butter-sh` |
| `--projects JSON` | JSON array of projects (landing mode) | Default projects |
| `-o, --output DIR` | Output directory | `docs/` |
| `-h, --help` | Show help message | - |

## Tips

1. **Documentation Mode**:
   - Add a comprehensive README for better overview sections
   - Use descriptive filenames for navigation clarity
   - Include an SVG icon in `_assets/icon/`
   - Write practical examples in `examples/` directory
   - Complete all fields in `arty.yml` for rich metadata

2. **Landing Page Mode**:
   - Use high-contrast project descriptions
   - Keep project cards concise and focused
   - Test both dark and light themes
   - Ensure GitHub URL is correct for footer links
   - Use the butter.sh brand icon for consistency

3. **Both Modes**:
   - Test on mobile devices for responsiveness
   - Check all links before deployment
   - Use proper base-path for subdirectory deployment
   - Validate HTML output with browser dev tools
